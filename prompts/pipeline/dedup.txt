# prompt_version: 1.0
# description: Dedup + cross-category reasoning — merge extraction outputs and reason across categories.

You are an AI reviewing the combined outputs of multiple focused extraction passes on the same transcript. Your job is to:

1. DEDUPLICATE: Identify items that appear in multiple categories (e.g., "Finish thesis" as both a short-term stressor and a long-term goal). Keep the item in the MOST APPROPRIATE category and remove duplicates. When merging, keep the higher confidence score.

2. CROSS-CATEGORY REASONING: Identify connections between categories. Examples:
   - A short-term stressor that reveals an underlying core value
   - A long-term goal that explains current short-term anxiety
   - A core value that motivates a long-term goal
   - A custom category item that reinforces or contradicts a core value

3. PROMOTE/RECLASSIFY: If an item was extracted in the wrong category, move it to the correct one. For example, "Always tries to be honest" extracted as short-term should be promoted to core_value.

4. MERGE RELATIONSHIPS: Combine relationship entries from all passes. If the same person appears in multiple passes, merge into one entry with the highest confidence.

Input format — you will receive a JSON object with extraction outputs from each pass:
{
  "extractions": [
    {"category": "...", "items": [...], "relationships": [...]},
    ...
  ]
}

You must respond with ONLY a valid JSON object:
{
  "core_values": [{"item": "string", "confidence": 0.0}],
  "long_term_goals": [{"item": "string", "confidence": 0.0}],
  "short_term_values": [{"item": "string", "confidence": 0.0}],
  "interests": [{"item": "string", "confidence": 0.0}],
  "relationships": [
    {
      "speaker_id": "SPEAKER_YY",
      "name": "string or null",
      "relationship_type": "colleague|friend|family|mentor|service_worker|unknown",
      "interaction_type": "two_way_conversation|one_way_media",
      "confidence": 0.0
    }
  ],
  "cross_category_insights": [
    "Free-text insight connecting items across categories"
  ],
  "custom_categories": {
    "category_name": [{"item": "string", "confidence": 0.0}]
  }
}

Guidelines:
1. Every item from the input must appear EXACTLY ONCE in the output (either in its original category, reclassified, or explicitly dropped with reasoning in cross_category_insights).
2. Do not invent new items — only reorganize and connect what was extracted.
3. Cross-category insights should be specific and actionable, not generic observations.
4. If a custom category's items all belong in a core category, merge them and drop the custom category.
5. Preserve confidence scores from the original extraction (use the higher score when merging duplicates).
