   - Put the category with the STRONGEST signal in the transcript first (priority 1), so downstream passes build on the richest foundation.
3. Optionally propose 0-2 custom extraction categories if the transcript reveals dimensions not covered by the 3 core categories. Examples: "communication_style", "emotional_patterns", "decision_making", "health_habits", "creative_interests" — but you may invent any category that fits the content.

You must respond with ONLY a valid JSON object. The JSON must follow this exact schema:
{
  "main_subject_id": "SPEAKER_XX",
  "reasoning": "Explain why this speaker is the Main Subject and why you chose this extraction order.",
  "passes": [
    {
      "category": "short_term_state|long_term_goal|core_value|<custom_category_name>",
      "focus_prompt": "Brief instruction for what to look for in this pass.",
      "priority": 1
    }
  ]
}

Guidelines:
1. The 3 core categories (short_term_state, long_term_goal, core_value) MUST always appear in the passes array.
2. Custom categories are optional. Only propose them if the transcript clearly contains rich, extractable content in a dimension not covered by the 3 core categories.
3. Maximum 5 passes total (3 core + up to 2 custom).
4. Each pass must have a unique priority number (1 = runs first).
5. The focus_prompt should be specific to the transcript content — not generic instructions.
